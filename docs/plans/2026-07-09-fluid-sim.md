# Scalable CA Fluid Simulation — Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers-extended-cc:executing-plans to implement this plan task-by-task.

**Goal:** Real flowing water, at any voxel scale, via a discrete swap-based cellular automaton stored directly in the existing chunk grid, with active-cell sleeping so settled water is free.

**Architecture:** A new `vox-sim` crate owns a `FluidSim` (an active-cell set, mirroring `PhysicsWorld`'s sleep bookkeeping) that ticks water voxels living in the real `vox_world::World` chunk storage — no separate fluid grid. A foundational fix in `vox-world` first makes `World::solid()` consult the material registry instead of hardcoding "solid = not air," additively (no existing call site breaks).

**Tech Stack:** Rust, existing workspace crates only (`vox-core`, `vox-world`). No new third-party dependencies.

**Design doc:** `docs/plans/2026-07-09-fluid-sim-design.md`

3. Add a new method near `solid()`:
   ```rust
   /// Attach a per-material-id solidity table (index = `Voxel.0`; build it
   /// from `MaterialRegistry` by mapping each id to its `MaterialDef.solid`).
   /// An id past the end of `table` reads as solid -- an out-of-range id
   /// should never happen in practice, and defaulting to *solid* is the
   /// safe failure direction (a stray id blocks movement/collision rather
   /// than silently letting the player walk through undefined material).
   pub fn set_solid_table(&mut self, table: Vec<bool>) {
       self.solid_table = Some(table);
   }

   /// True when material id `id` counts as solid, consulting the attached
   /// table if there is one. Shared by `solid()` and `SolidLookup::solid()`
   /// so the two can never disagree.
   fn material_is_solid(&self, id: u16) -> bool {
       match &self.solid_table {
           Some(table) => table.get(id as usize).copied().unwrap_or(true),
           None => true, // legacy fallback: any non-air voxel is solid
       }
   }
   ```

   ```rust
   pub fn solid(&mut self, v: IVec3) -> bool {
       if !self.world.in_bounds(v) {
           return false;
       }
       let key = chunk_of(v);
       let chunk = match self.cached {
           Some((k, c)) if k == key => c,
           _ => {
               let c = self.world.chunk_at(key);
               self.cached = Some((key, c));
               c
           }
       };
       match chunk.and_then(|c| {
           let voxel = c.get(local_of(v));
           (voxel != AIR).then_some(voxel)
       }) {
           Some(voxel) => self.world.material_is_solid(voxel.0),
           None => false,
       }
   }
   ```

```rust
//! `docs/plans/2026-07-09-fluid-sim-design.md` for the rationale.

use glam::IVec3;
use vox_core::FxHashSet;

/// Tracks which water cells are still moving. A cell not in this set is
/// settled and costs nothing to tick -- the entire performance story of
/// this crate (mirrors `PhysicsWorld`'s sleep bookkeeping).
pub struct FluidSim {
    active: FxHashSet<IVec3>,
    /// xorshift64* state for randomized per-tick update order (same
    /// construction as `PhysicsWorld::lifetime_rng` / `ParticleSystem`'s
    /// spawn jitter) -- avoids a visible left/right or diagonal bias in how
    /// water spreads.
    rng: u64,
}

impl Default for FluidSim {
    fn default() -> Self {
        Self::new()
    }
}

impl FluidSim {
    pub fn new() -> Self {
        Self {
            active: FxHashSet::default(),
            rng: 0x9E37_79B9_7F4A_7C15,
        }
    }

    /// Number of cells currently flowing (debug-overlay stat).
    pub fn active_count(&self) -> usize {
        self.active.len()
    }

    /// xorshift64* -- deterministic, dependency-free.
    fn next_u64(&mut self) -> u64 {
        self.rng ^= self.rng << 13;
        self.rng ^= self.rng >> 7;
        self.rng ^= self.rng << 17;
        self.rng
    }
}

}
```

```rust
    /// Fill a sphere of radius `radius_vox` centered on `center` with
    /// `water_material`, skipping any cell that isn't currently air (never
    /// carves through existing terrain), and mark every filled cell active.
    /// `radius_vox` is in voxels, not meters -- the caller (the placement
    /// tool) converts from the tool's meter-radius via the world's
    /// `voxel_size_m`, same convention as `vox_physics::carve_sphere`.
    pub fn place_blob(
        &mut self,
        world: &mut World,
        center: IVec3,
        radius_vox: i32,
        water_material: Voxel,
    ) {
        let r = radius_vox.max(0);
        let min = center - IVec3::splat(r);
        let max = center + IVec3::splat(r + 1);
        let r2 = (r * r) as i64;
        let mut filled = Vec::new();
        world.edit_box(min, max, |v, cur| {
            if cur != AIR {
                return None;
            }
            let d = v - center;
            let dist2 = (d.x as i64 * d.x as i64) + (d.y as i64 * d.y as i64) + (d.z as i64 * d.z as i64);
            if dist2 > r2 {
                return None;
            }
            filled.push(v);
            Some(water_material)
        });
        self.active.extend(filled);
    }
```

```rust
/// Where a water cell at `pos` wants to move this tick, or `None` if it has
/// nowhere to go (should settle). `is_open` reports whether a cell can be
/// flowed into: empty (air) and in-bounds. Order of preference: straight

/// front/back) -- see the design doc §4 for why this shape and not
/// fractional pressure levels.
fn step_cell(pos: IVec3, is_open: &mut impl FnMut(IVec3) -> bool, coin: bool, coin2: bool) -> Option<IVec3> {
    let down = pos + IVec3::NEG_Y;
    if is_open(down) {
        return Some(down);
    }

    let (dx1, dx2) = if coin { (1, -1) } else { (-1, 1) };
    for dx in [dx1, dx2] {
        let diag = pos + IVec3::new(dx, -1, 0);
        if is_open(diag) {
            return Some(diag);
        }
    }
    let (dz1, dz2) = if coin2 { (1, -1) } else { (-1, 1) };
    for dz in [dz1, dz2] {
        let diag = pos + IVec3::new(0, -1, dz);
        if is_open(diag) {
            return Some(diag);
        }
    }

    // Sideways spread requires support directly below the destination --
    // otherwise this degenerates into the diagonal case, which already ran.
    let (sx1, sx2) = if coin { (1, -1) } else { (-1, 1) };
    for dx in [sx1, sx2] {
        let side = pos + IVec3::new(dx, 0, 0);
        if is_open(side) && !is_open(side + IVec3::NEG_Y) {
            return Some(side);
        }
    }
    let (sz1, sz2) = if coin2 { (1, -1) } else { (-1, 1) };
    for dz in [sz1, sz2] {
        let side = pos + IVec3::new(0, 0, dz);
        if is_open(side) && !is_open(side + IVec3::NEG_Y) {
            return Some(side);
        }
    }

    None
}
```

And on `impl FluidSim`:

```rust
    /// Advance every active cell by one tick: move it per `step_cell`, or
    /// drop it from the active set if it has settled. A cell that moves
    /// reactivates itself, its destination, and the destination's neighbors
    /// -- the entire wake cascade, no separate propagation pass needed.
    pub fn tick(&mut self, world: &mut World) {
        let water = self
            .active
            .iter()
            .next()
            .map(|&p| world.get_voxel(p))
            .unwrap_or(AIR);
        if water == AIR {
            return; // nothing active, or (shouldn't happen) it already drained
        }

        let cells: Vec<IVec3> = self.active.iter().copied().collect();
        let mut next_active = FxHashSet::default();
        for pos in cells {
            if world.get_voxel(pos) != water {
                continue; // moved/consumed by an earlier cell this same tick
            }
            let coin = self.next_u64() & 1 == 0;
            let coin2 = self.next_u64() & 1 == 0;
            let mut is_open = |p: IVec3| world.in_bounds(p) && world.get_voxel(p) == AIR;
            let dest = step_cell(pos, &mut is_open, coin, coin2);
            match dest {
                Some(dest) => {
                    world.set_voxel(pos, AIR);
                    world.set_voxel(dest, water);
                    next_active.insert(dest);
                    for n in NEIGHBORS_6 {
                        next_active.insert(dest + n);
                    }
                }
                None => {} // settled -- not re-added
            }
        }
        self.active = next_active;
    }
```

```rust
        let Some(water_id) = registry.id_by_name("water") else {
            return; // asset set doesn't define water; nothing to place
        };
        let s = world.cfg.voxel_size_m;
        let center_vox = vox_core::voxel_at(eye_m + dir * hit.dist_m, s);
        let radius_vox = (self.radius_m / s).round() as i32;
        fluid.place_blob(world, center_vox, radius_vox, Voxel(water_id.0));
    }
```

`voxel_at` is re-exported at `vox_core`'s crate root (`pub use coords::{..., voxel_at, ...}` in `vox-core/src/lib.rs`) — no need to go through the `coords` submodule path. Add it to this file's existing `use vox_core::{MaterialRegistry, voxel_center_m};` import line.

**Step 2: Write the test**

In `crates/vox-app/src/tools.rs`'s test module:

```rust
#[test]
fn digging_into_a_settled_lake_lets_it_flow_into_the_new_opening() {
    let reg = registry_with_water();
    let mut world = World::new(WorldConfig {
        voxel_size_m: 1.0,
        extent_m: [32.0, 32.0, 32.0],
        ..WorldConfig::default()
    });
    world.set_solid_table(solid_table_for(&reg));
    let (_, max) = world.bounds_voxels();
    world.fill_box(IVec3::ZERO, IVec3::new(max.x, 10, max.z), stone_id(&reg));
    // A basin: a walled depression on top of the floor, so water placed
    // inside it settles instead of just spreading everywhere.
    world.fill_box(IVec3::new(10, 10, 10), IVec3::new(20, 13, 20), Voxel(0)); // hollow
    world.fill_box(IVec3::new(9, 10, 9), IVec3::new(21, 13, 21), stone_id(&reg)); // walls (overwritten below)
    world.fill_box(IVec3::new(10, 10, 10), IVec3::new(20, 13, 20), Voxel(0)); // re-hollow the interior

    let mut sim = FluidSim::new(water_voxel(&reg));
    let tools = Tools::new(&reg);
    tools.place_water(&mut world, &mut sim, &reg, Vec3::new(15.0, 20.0, 15.0), Vec3::new(0.0, -1.0, 0.0));
    for _ in 0..200 {
        sim.tick(&mut world);
    }
    assert_eq!(sim.active_count(), 0, "the basin's lake must settle");
    let water_before = (10..20)
        .flat_map(|x| (10..20).map(move |z| (x, z)))
        .filter(|&(x, z)| world.get_voxel(IVec3::new(x, 11, z)) == water_voxel(&reg))
        .count();
    assert!(water_before > 0, "the basin must actually hold water before the test digs into it");

    // Dig a one-voxel hole through the basin's east wall at water height.
    let breach = IVec3::new(20, 11, 15);
    world.set_voxel(breach, Voxel(0));
    sim.wake_region(&world, breach - IVec3::ONE, breach + IVec3::splat(2));
    assert!(sim.active_count() > 0, "digging into the lake must wake it");

    for _ in 0..100 {
        sim.tick(&mut world);
    }
    assert_eq!(
        world.get_voxel(IVec3::new(21, 11, 15)),
        water_voxel(&reg),
        "water must have flowed out through the breach"
    );
}
```

Adjust the basin geometry if the first `fill_box` calls turn out to overwrite each other in an unintended order — verify with a quick debug print or a smaller intermediate assertion before trusting the final one.
